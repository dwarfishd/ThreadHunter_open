## Реальные проблемы (3) — доказаны и починены

| # | Проблема | Почему реально для НАШЕГО проекта | Фикс |
|---|---|---|---|
| 1 | `th init` хардкодит путь к БД в выводе | Менеджер задаёт TH_DB=/Volumes/USB/db для бэкапа (T5: «один файл — одна точка отказа»). Видит неверный путь, не может найти файл. | `get_db_path()` вместо хардкода в cli.py |
| 2 | Crash если `~/.threadhunter` — файл | Целевая аудитория — не-технические менеджеры. `touch ~/.threadhunter` по ошибке → crash без объяснения. Потеря доверия. | Проверка `parent.is_dir()` перед `mkdir` |
| 3 | `load_dotenv(override=False)` не перезаписывает env vars | Если пользователь сделал `export API_ID=` в `.zshrc` (редко, но бывает), .env игнорируется. | `override=True` |

## Что починили в методологии

- **«Это РЕАЛЬНО проблема нашего проекта?»** — каждый баг проверять через: аудитория (менеджер, не dev), объёмы (десятки, не тысячи), приоритет (запустить → продать → масштабировать). Теоретические проблемы для фазы 5+ не чинить превентивно.
- load_dotenv: всегда `override=True`

## Что в долгую память

- `load_dotenv(override=True)` — не False
- FK в SQLite работает после executescript
- `_LOG_DIR.exists() and not _LOG_DIR.is_dir()` — защита от файла вместо директории
