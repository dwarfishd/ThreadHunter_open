name: 5 moves after feature
description: Цикл из 5 ходов после каждой фичи — errors, gaps, reality check, fresh view, security
type: feedback
---

После завершения каждой фичи обязательно выполнять цикл из 5 ходов:

1. **Найти ошибки** — доказать, решить.
2. **Что упустили** — доказать, что не покрыто.
3. **Это РЕАЛЬНО проблема нашего проекта?** — отфильтровать теоретические риски от реальных.
4. **Свежий взгляд после правок** — перечитать код как новый человек.
5. **Security review** — опасная тройка: личные данные + чужой контент + отправка наружу.

**Why:** Без системной проверки после фичи ошибки, пробелы и security-риски уходят в main. Цикл гарантирует минимальное качество.

**How to apply:** После каждого `git commit` фичи — не переходить к следующей задаче, а пройти все 5 шагов. Документировать результаты. Для micro-фич можно упростить, но security review (шаг 5) обязателен всегда.
