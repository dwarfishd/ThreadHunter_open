Пользователь — создатель и менеджер проекта ThreadHunter. Занимается проектированием архитектуры, определением требований и принятием технических решений. Работает над CLI-утилитой для парсинга Telegram-каналов швейных производств (Бишкек, Россия). Интересуется локальным хранением данных, AI-классификацией и минимализмом в зависимостях.

**Why:** Понимание роли помогает выбирать глубину объяснений — пользователь принимает архитектурные решения, а не просто пишет код.

**How to apply:** При предложениях по архитектуре давать контекст «почему», а не только «как». Избегать излишних объяснений базовых концепций.
