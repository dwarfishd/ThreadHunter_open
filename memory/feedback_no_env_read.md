---
name: No env read
description: .env только через grep "^VAR=", не Read целиком
type: feedback
---

Работать с `.env` только через `grep "^VAR=" .env` для получения конкретного значения. Никогда не читать `.env` целиком через Read или cat.

**Why:** `.env` содержит секреты (API-ключи, токены). Чтение целиком может засветить их в контексте разговора или логах.

**How to apply:** Когда нужна конкретная переменная — использовать `grep "^VAR_NAME=" .env`. Если нужен обзор — спрашивать пользователя, какие переменные существуют, не читая файл.
