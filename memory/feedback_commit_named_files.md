---
name: Commit named files
description: git add только поимённо, никогда . или -A
type: feedback
---

`git add` только поимённо — каждый файл должен быть указан явно. Никогда не использовать `.` или `-A`.

**Why:** `.` и `-A` захватывают временные файлы, логи, артефакты сборки, которые не должны попасть в коммит. Поимённый add осознанно контролирует состав коммита.

**How to apply:** При подготовке коммита перечислять каждый файл отдельно: `git add file1.py file2.py`. Перед `git commit` проверять `git status` — только intended changes.
