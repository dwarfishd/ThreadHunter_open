# Local First

## Принцип

**Данные и обработка — локально.** Пользователь не зависит от облачного сервера, не платит за хостинг, не рискует утечкой данных. Всё работает на его машине.

## Что это значит

- **Данные хранятся локально** — SQLite или локальные файлы. Нет облачной базы, нет синхронизации через сервер.
- **Парсинг работает локально** — парсер запускается на устройстве пользователя. Нет удалённых воркеров, нет очередей, нет сторонних сервисов.
- **Полная автономность** — продукт работает без интернета (кроме доступа к Telegram API). Отключился сервер — ничего не сломалось.

## Почему это важно

- **Приватность** — спарсенные контакты и данные не уходят на чужой сервер.
- **Контроль** — пользователь владеет своей базой целиком. Может делать бэкапы, переносить, удалять.
- **Дешевизна** — нет расходов на серверную инфраструктуру. Продукт работает на том, что уже есть у пользователя.
- **Простота архитектуры** — один процесс, одна машина, одна база данных. Меньше точек отказа.

## Как применять

- При выборе между локальным и облачным решением — выбирать локальное.
- Не проектировать синхронизацию, мульти-юзер аккаунты, веб-интерфейс.
- Бэкапы — экспорт файлов, не облачный sync.
- Если в будущем понадобится масштабирование — думать о локальных копиях с ручным мёржем, а не о переходе в облако.
