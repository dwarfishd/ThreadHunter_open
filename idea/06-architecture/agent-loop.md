# Agent Loop

## Цикл обработки

Парсинг Telegram-каналов с AI-классификацией извлечённых данных. Цикл работает по расписанию, без участия пользователя.

## Шаги цикла

### 1. Trigger (по расписанию)
- Пользователь настроил частоту: раз в день / неделю / месяц
- Системный таймер запускает цикл
- Нет ручного запуска — пользователь приходит к готовым данным

### 2. Fetch (парсинг каналов)
- Подключение к Telegram API с локальными credentials пользователя
- Чтение новых сообщений из настроенных каналов
- Извлечение текста, контактов, дат, вложений
- Дедупликация: пропуск уже спарсенных постов

### 3. Extract (извлечение сущностей)
- NER (Named Entity Recognition): имена, телефоны, города, компании
- Извлечение контактов из текста (телефон, Telegram-username, email)
- Определение дат, объёмов, сроков

### 4. Classify (AI-классификация)
- Определение тегов:
  - **Локация** — город, регион (Бишкек, Москва, Ош и т.д.)
  - **Ассортимент** — что шьют (футболки, куртки, спецодежда и т.д.)
  - **Тип предложения** — ищут подрядчика / предлагают услуги / ищут сотрудников
- Локальная LLM или лёгкий модельный пайплайн (без облачных вызовов)

### 5. Store (сохранение)
- Запись в локальную SQLite-базу
- Каждый пост = одна запись с полями: текст, контакты, теги, дата, источник
- Логирование ошибок парсинга

### 6. Notify (опционально)
- Если появились новые объявления, соответствующие интересам пользователя — уведомление
- Не спамить: группировка по каналам, показ только нового

## Диаграмма

```
[Timer] → [Fetch Telegram] → [Extract Entities] → [AI Classify] → [Store SQLite] → [Notify?]
                ↑                                                        |
                └──────────────────── Schedule ──────────────────────────┘
```
