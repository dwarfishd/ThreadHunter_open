# Triggers

## Запуск цикла обработки

Цикл запускается автоматически по расписанию, без ручного вмешательства.

## Типы триггеров

### 1. Scheduled (основной)
- **Daily** — раз в день (по умолчанию). Оптимально для большинства каналов.
- **Weekly** — раз в неделю. Для каналов с низкой частотой публикаций.
- **Monthly** — раз в месяц. Для архивного сбора или тестирования.

### 2. Manual (override)
- Пользователь может запустить парсинг вручную через CLI
- Используется для: первичного заполнения базы, отладки, форс-мажоров
- Не отменяет следующий scheduled запуск

### 3. On-demand channel add
- При добавлении нового канала — автоматический один раз парсинг этого канала
- Затем канал включается в регулярное расписание

## Реализация

- **Unix-like**: cron job или systemd timer
- **macOS**: launchd plist
- **Windows**: Task Scheduler
- **Fallback**: собственный демоном/сервис, который работает в фоне

## Обработка ошибок

- Если парсинг упал — лог ошибки, retry через 1 час (максимум 3 retry)
- Если все retry失敗 — уведомление пользователю при следующем запуске приложения
- Не блокировать следующий scheduled запуск из-за прошлого failure
