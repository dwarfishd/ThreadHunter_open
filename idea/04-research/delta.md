# Delta: чем идея уникальна

## Вертикальная специализация

**Что никто не делает:** ни один из 30+ исследованных инструментов не парсит Telegram-каналы specifically для швейной/производственной ниши.

- **Web3/crypto — доминирующая вертикаль** (CRMChat, Spredo, Enreach, DISE CRM). Все остальные парсеры — общего назначения (Tegranium, TGCParser, Apify).
- **B2B-директории** (Kompass, ThomasNet, Alibaba, Textile Pages) покрывают глобальные рынки или Азию (Китай, Индия, Бангладеш, Вьетнам). **Ни одна не покрывает СНГ/Центральную Азию.**
- **Швейная ниша Бишкека/России — пустое место.** Нет директории, нет парсера, нет базы лидов.

**Почему это моат:** вертикальная специализация создаёт domain-specific tagging schema, которую general-purpose парсеры не воспроизведут без инвестиций в экспертизу. Категоризация по типу бизнеса (фабрика/цех/подрядчик), ассортименту (одежда/обивка/промтекстиль), локации (Бишкек/Ош/Москва) — это не просто теги, это структура, которая делает данные полезными.

## Тегирование по локации и ассортименту

**Что делают конкуренты:** извлекают сырые данные (юзернеймы, телефоны, текст сообщений). Без тегирования.

**Что делает идея пользователя:** каждый лид получает структурированные теги:
- **Локация:** Бишкек, Ош, Москва, и т.д. (извлекается через spaCy NER + keyword matching)
- **Ассортимент:** одежда, обивка, промтекстиль, аксессуары (domain-specific keyword matching)
- **Тип бизнеса:** фабрика, цех, подрядчик, поставщик (NER + rule-based классификация)
- **Тип контакта:** телефон, email, Telegram handle, сайт (из message entities Telegram)

**Почему это моат:** сырые данные Telegram бесполезны без структурирования. Тегирование — это то, что превращает хаос в базу. General-purpose парсер не знает, что «пошив оптом» и «швейный цех» — это одна категория, а «кожаная обивка» — другая.

## Реальная активность вместо статичных профилей

**Что делают директории:** статичные профили компаний (название, адрес, категория). Не обновляются месяцами.

**Что делает идея пользователя:** парсинг Telegram-каналов captures то, что фабрики **прямо сейчас рекламируют**. Каждое сообщение — сигнал активности: какие продукты предлагаются, какие контакты актуальны, какие мощности доступны.

**Почему это моат:** static directories показывают кто существует. Telegram-парсер показывает кто **активен**. Это фундаментально другой тип данных, который не может быть воспроизведён crawler'ом статичных сайтов.

## Русский язык

**Что делают конкуренты:** большинство B2B-платформ (Apollo, ZoomInfo, Clay, Lusha) — English-first, без русской локализации.

**Что делает идея пользователя:** русскоязычный интерфейс, русскоязычная NER (spaCy `ru_core_news_lg`), ключевые слова на русском. Целевые пользователи (менеджеры, ищущие швейные фабрики в Бишкеке/России) работают преимущественно на русском.

**Почему это моат:** Western players не будут инвестировать в русскоязычную NER и domain-specific keywords для швейной ниши СНГ. Это достаточно маленький рынок, чтобы быть непривлекательным для глобальных игроков, но достаточно большой для нишевого SaaS.

## Ценовая ниша $10-50/мес

**Что существует:**
- B2B lead-gen платформы: $49-150/пользователь/мес (Apollo, Lusha, Clay)
- Enterprise B2B data: $15K+/год (ZoomInfo, Cognism)
- Telegram-инфраструктура: ~$49/мес + usage (Apify)
- Бесплатные/open-source парсеры: $0, но требуют технических навыков

**Что свободно:** ни один инструмент не берёт $10-50/мес за вертикальную базу лидов из Telegram. Это ценовая точка, доступная для малого/среднего бизнеса в СНГ, но слишком низкая для enterprise-платформ.

**Почему это моат:** низкая цена + вертикальная специализация = высокий switching cost для пользователей, которые уже инвестировали в тегирование и структурирование своих данных. Переход на general-purpose парсер означает потерю domain-specific структуры.

## Сводка: что никто не делает и почему это моат

| Уникальный элемент | Кто пытается | Почему не получится у них |
|---|---|---|
| Вертикаль: швейная ниша Бишкека/России | Никто | Western players: слишком маленький TAM. Локальные игроки: нет технических компетенций |
| Domain-specific tagging schema | Никто | General-purpose парсеры не инвестируют в niche taxonomy |
| Real-time activity signal из Telegram | Tegranium, TGCParser (но не вертикальны) | У них нет domain knowledge для структурирования швейных лидов |
| Русскоязычная NER + UI | Tegranium, TGStat, Telemetr (русские основатели) | У них нет фокуса на швейную нишу и тегирование |
| Цена $10-50/мес за niche lead DB | Никто | Enterprise-платформы не опустятся, free tools не дадут структурирование |
