# 04-research — TL;DR

*Дата: 2026-06-30*

## Что нашли

Пять источников (социальные тренды, SaaS-лендскейп, акселераторы, open-source, конкуренты) рисуют一致ную картину: **рынок парсеров Telegram существует, фрагментирован, но ни один игрок не сделал вертикально-специфичную базу лидов для швейной/производственной ниши.**

### Кратко по каждому источнику

| Источник | Главное |
|---|---|
| **social-trends.md** | 8-10 коммерческих инструментов, ни один не доминирует. Web3/crypto — доминирующая вертикаль. Швейная ниша — пустое место. Chrome-экстеншены — самый низкий порог входа. |
| **saas-landscape.md** | Прямых конкурентов в вертикали нет. TGStat/Telemetr имеют инфраструктуру для пивота, но пока не экстрагируют контакты. B2B-директории (Kompass, ThomasNet) не покрывают СНГ/Центральную Азию. Ценовая ниша $10-50/мес свободна. |
| **accelerators-landscape.md** | Паттерн «неструктурированные данные → структурированные записи» валидирован (DryMerge W24, Segment S11). YC не финансирует CIS-регион. Красный флаг: platform dependency risk (API Telegram). |
| **open-source-landscape.md** | Telethon (~8k stars) + tg-archive (~500 stars) + NocoDB (~50k stars) = готовая архитектура MVP за 2-4 недели. spaCy с русской NER-моделью — для извлечения локаций/компаний. python-phonenumbers — для телефонов. |
| **competitors-landscape.md** | CRMChat ($2.5M raised, $440K ARR) — ближайший финансируемый игрок, но это CRM-оверлей, не парсер/база. Tegranium — ближайший прямой конкурент (AI-парсер + CRM, русский язык), но не вертикален. |

## 3-5 ключевых выводов

1. **Дельта идеи — вертикальная специализация.** Ни один из 30+ исследованных инструментов не парсит Telegram-каналы specifically для швейной/производственной ниши Бишкека/России. Все существующие парсеры — общего назначения или Web3-ориентированные.

2. **Ценность не в скрапинге, а в структурировании.** Все источники подтверждают: сырые данные Telegram требуют значительной очистки. Моат — в тегировании по локации, ассортименту, типу бизнеса и контактам, а не в самом парсинге.

3. **Готовая архитектура MVP.** Telethon → python-phonenumbers + spaCy NER → SQLite → NocoDB UI. Open-source компоненты MIT/Apache-лицензированы, оценка MVP — 2-4 недели.

4. **Потенциальные угрозы пивота — средние.** TGStat/Telemetr (имеют данные), CRMChat (имеют пользователей), Apify (имеют инфраструктуру) — никто пока не фокусируется на производстве. Окно ~12-18 месяцев до возможного пивота.

5. **Ценовая ниша $10-50/мес свободна.** Существующие B2B-платформы стоят $49-150/пользователь/мес или $15K+/год. Ни один инструмент не берёт премию за «вертикальный Telegram-парсер».
