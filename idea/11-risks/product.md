# Product Risks — Post-Mortem

## P1. Отсутствие острого Buyer Pain

**Уровень: CRITICAL**

Менеджеры швейных производств находили подрядчиков через WhatsApp-группы, сарафанное радио и личные контакты. Парсер Telegram-каналов решал nice-to-have, а не must-have. Без острой боли продукт не используется — даже если он бесплатный.

**Цена:** 0 retention. Пользователи попробовали раз, не нашли ценности, забыли.

**Что надо было сделать:** До первого коммита: 15 интервью с менеджерами, показать мокап, взять предоплату. Если никто не дал $10 — идеи не существует. Валидация спроса ДО разработки, не после.

## P2. CLI отпугнул целевую аудиторию

**Уровень: HIGH**

Менеджеры швейных цехов в Бишкеке и регионах России не открывают терминал. Они работают с телефона, через WhatsApp и Telegram. `python main.py parse --daily` — это не интерфейс для них, это стена.

**Цена:** 95% целевой аудитории отсеялись на этапе установки. Из 100 человек, которым показали продукт, 95 не смогли его запустить.

**Что надо было сделать:** Telegram-бот как интерфейс. Пользователь пишет «найди швей в Бишкеке» — получает 10 контактов. Без терминала, без Python, без setup. CLI — для разработчиков, бот — для менеджеров.

## P3. Качество данных не оправдало ожиданий

**Уровень: HIGH**

Сырые данные Telegram + локальная LLM = много шума. Дубликаты, нераспознанные контакты, ложные теги по локации и ассортименту. Пользователь получил 50 «найденных» контактов, из которых 20 — нерабочие, 15 — не по профилю, 10 — дубли.

**Цена:** Доверие сломалось после первого использования. Пользователь решил «данные плохие» — и больше не вернулся.

**Что надо было сделать:** MVP: минимальный набор полей, но высокое качество. Регулярки для телефонов (90%+ точность), гео-справочник для городов. Показывать source, дату, статус каждого объявления. Честно коммуницировать: это инструмент для поиска, а не готовая CRM.

## P4. Неправильная модель монетизации

**Уровень: MEDIUM**

Подписка $10-50/мес за доступ к базе парсенных контактов — пользователь не понимает, за что платит. Данные из Telegram «бесплатные» (он может сам читать каналы). Ценность не очевидна.

**Цена:** 3 trial пользователя → 0 конверсий в paying. churn до первого биллинга.

**Что надо было сделать:** Pay-per-result: пользователь платит за каждый рабочий контакт, который он получил. Или freemium: 50 контактов бесплатно, дальше $0.10/контакт. Ценность должна быть измеримой.

## P5. Узкий MVP — недостаточно ценности для удержания

**Уровень: MEDIUM**

MVP = парсинг + сортировка + базовая аналитика. Без экспорта, без CRM-интеграций, без outreach. Пользователь получил данные — и что дальше? Ручной обзвон? Ручное копирование в Excel?

**Цена:** Пользователи получили данные, но не знали, что с ними делать. Нет workflow → нет привычки → нет retention.

**Что надо было сделать:** CSV/Excel экспорт — день разработки, но закрывает последний шаг workflow. Или: одна кнопка «отправить контакт в WhatsApp» — и продукт становится action-oriented, как заявлено в принципах.
