# Trade-offs

## Сознательные компромиссы

### Локальная LLM ↔ Качество классификации
**Теряем:** Точность классификации ниже, чем у облачных моделей (GPT-4, Claude). Локальные 7B-модели хуже понимают контекст, могут путать теги.
**Выигрываем:** Полная приватность, ноль облачных зависимостей, соответствует Local First.
**Митигация:** Можно дообучить модель на размеченных данных швейной ниши. Или дать пользователю возможность ручной корректировки тегов.

### CLI ↔ Accessibility
**Теряем:** Часть аудитории, которая боится командной строки. Менеджер без технической подготовки может не освоить.
**Выигрываем:** Скорость разработки (x3-x5 быстрее, чем GUI), простота поддержки, лёгкая автоматизация.
**Митигация:** CLI команды минимальны (5 штук). Можно добавить `--help` с примерами. GUI — фаза 2, если продукт подтвердит ценность.

### SQLite ↔ Масштабируемость
**Теряем:** Нельзя масштабировать до миллионов записей. Нет конкурентного доступа, нет репликации.
**Выиграваем:** Ноль конфигурации, один файл, работает сразу после установки.
**Митигация:** Для MVP и целевой аудитории (тысячи постов, не миллионы) — SQLite достаточно. Если вырастет — миграция на PostgreSQL.

### Системные планировщики ↔ Кроссплатформенная сложность
**Теряем:** Нужно поддерживать 3 разных механизма (cron, launchd, Task Scheduler). Нет единого кода.
**Выигрываем:** Надёжность, переживание перезагрузки, не свой демон.
**Митигация:** Обернуть в абстракцию `scheduler.py` с тремя бэкендами. Документация для каждой ОС.

### Сохранять всё ↔ Качество данных
**Теряем:** База содержит «шум» — посты без контактов, дубликаты, нерелевантный контент.
**Выигрываем:** Ничего не потеряно. Пользователь видит полную картину и сам фильтрует.
**Митигация:** Фильтры и теги в CLI. Команда `filter --has-contacts`. Возможность ручной маркировки.

### Telegram API только для чтения ↔ Функциональность
**Теряем:** Не можем отправлять сообщения, отмечать посты как «связался», взаимодействовать с каналом.
**Выигрываем:** Проще архитектура, меньше прав, меньше риск блокировки.
**Митигация:** Экспорт контактов → пользователь сам связывается. Отметки о статусе — локально в базе.

### Нет мульти-юзера ↔ TAM
**Теряем:** Не продаём командам и отделам. Ограниченный TAM.
**Выигрываем:** Простая архитектура, быстрая разработка, соответствует One Person Company.
**Митигация:** Один менеджер = один покупатель продукта. Если команда из 5 менеджеров — 5 лицензий. Или фаза 2: мульти-юзер.
