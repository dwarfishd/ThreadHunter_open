# Product Decisions

## Принятые продуктовые решения

### 1. Сохранять посты даже без контактов
**Решение:** Каждый спарсенный пост сохраняется в базу, даже если AI не нашёл контактов.

**Альтернативы:**
- Выбрасывать посты без контактов — чище база, но теряется контекст

**Почему:** Пост может содержать полезную информацию (например, описание компании, объёмы), даже если контакт не распознан. Пользователь сам отфильтрует. Соответствует MVP-принципу: «собираем всё, потом отфильтруем».

### 2. CLI, не GUI
**Решение:** Командная строка — единственный интерфейс.

**Альтернативы:**
- Desktop GUI (Electron, Tauri) — красивее, но тяжелее
- Web UI — нарушает Local First

**Почему:** CLI быстрее в разработке, легче в поддержке, легко автоматизировать. Целевая аудитория — менеджер, который может освоить 5 команд. GUI добавит 3x к объёму работы.

### 3. Парсинг по расписанию, не по запросу
**Решение:** Система работает автоматически. Пользователь не нажимает «запустить».

**Альтернативы:**
- Только ручной запуск — проще технически, но нарушает Proactive

**Почему:** Соответствует принципу Proactive. Менеджер — не оператор парсера. Он приходит к готовым данным.

### 4. Один пользователь, одна база
**Решение:** Нет мульти-юзера, нет общих баз, нет ролей.

**Альтернативы:**
- Командный доступ — больше TAM, но сложнее архитектура

**Почему:** Соответствует принципу One Person Company. Продукт для менеджера-одиночки, не для отдела. Масштабирование = больше данных, не больше пользователей.

### 5. Теги: локация + ассортимент + тип предложения
**Решение:** Три типа тегов в MVP. Остальные — позже.

**Альтернативы:**
- Больше тегов сразу (объёмы, сроки, цены) — богаче данные, но сложнее классификация

**Почему:** Три типа покрывают 80% use case: «найти швейное производство в Бишкеке, которое ищет подрядчика». Остальное — фаза 2.

### 6. Экспорт CSV как первая интеграция
**Решение:** После MVP — экспорт в CSV/Excel, не CRM API.

**Альтернитивы:**
- Прямая интеграция с CRM — мощнее, но сложнее и раньше времени

**Почему:** CSV работает с любой CRM. Менеджер сам импортирует куда хочет. Ноль зависимостей, ноль API-ключей.
