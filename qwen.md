## Деплой

GitHub Releases — PyInstaller, бинарники для macOS / Windows / Linux. **Деплоить только по моей явной команде.**

## 8 правил репо

1. Не хардкодить динамику (цены, даты, ID, ссылки) — из БД или конфига.
2. Не деплоить без моей явной команды.
3. Удалять одноразовый код после использования.
4. Перед `git push` — `npm run build`, не только `tsc`.
5. Большая фича = план в `plans/`. Микрофича — без.
6. После каждой фичи — цикл из 5 ходов.
7. `git add` поимённо. Никогда `.` или `-A`.
8. `.env` только через `grep "^VAR=" .env`.

## Большая фича = план

- Один план = одна фича.
- Имя: `YYYY-MM-DD-имя.md`.
- Фазы со статусами `[ ]` / `[WIP]` / `[x]`.
- По фазе: цель / deliverable / критерий «сделано» / задачи.
- В конце — итог: реализован, что осталось, коммиты.

## Цикл из 5 ходов после фичи

1. Найди ошибки → докажи → реши.
2. Что упустили → докажи.
3. Это РЕАЛЬНО проблема нашего проекта?
4. Свежий взгляд после правок.
5. Security review (опасная тройка: личные данные + чужой контент + отправка наружу).
